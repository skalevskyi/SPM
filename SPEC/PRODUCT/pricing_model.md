# Pricing Model — ADMOVE

**Pricing is determined dynamically using an estimation model.** The website uses a calculator to generate indicative pricing. This document describes the pricing estimation concept, possible variables, and future calculator logic. **It does not contain fixed prices.**

## Pricing Estimation Concept

- Prices are **not** fixed in a static ladder for public display.
- Indicative pricing is generated by an **estimation model** (e.g. format, duration, optional parameters).
- The landing page in **Phase 2A** shows a **calculator placeholder only** (expandable UI, static fields, CTA to contact).
- **Phase 2B** will implement **real calculator logic** that produces indicative quotes based on the estimation model.

## Possible Variables (for future calculator)

Variables that may feed into the estimation model include (non-exhaustive):

- **Format:** Rear, Side, Full (different exposure and production implications).
- **Duration:** Campaign length (e.g. 1 month, 3 months, 6 months).
- **Production:** Design and production of creatives (vinyl, print, application) — may be included or invoiced separately; to be defined in commercial policy.
- **Commitments:** Minimum duration and possible discounts for longer commitments (commercial policy).

Exact formula and commercial rules (units, discounts, production inclusion) are to be defined in Phase 2B and commercial documentation. The website and any quote tool must align with the same estimation logic once defined.

## CPM (internal / B2B)

- **CPM** (cost per thousand contacts) can be derived from visibility (see visibility_model.md) and estimated price when needed.
- Used internally or in B2B arguments; display on the site is optional and not part of Phase 2A.

## Contract and Production

- Minimum duration and possible commitments to be defined in commercial policy.
- Production costs: either included in the format estimate or invoiced separately; website and commercial documents should state which applies.
